# Level

A multi-purpose **horizontal level**, which can contain almost any other element.

The structure of a `<Level>` is:

- `<Level>`: main container
  - `<Level.Item>`: for each individual element
    - `<Level.Item>`

In a `<Level.Item>`, you can insert almost _anything_ you want: a `<Title>`, a `<Button>`, an `<Input>`, or just text.
Regardless, these elements will always be **vertically centered**.

### Centered level

If you don't use the `align` prop of `<Level.Item>`, you can include as many `<Level.Item>`s and they'll remain centered – as long as they're the **direct** descendent of the `<Level>`.

### Breakpoint

Use the `breakpoint` prop of `<Level>` to change the viewport size at which `<Level.Item>`s are vertically placed rather than horizontal.

For example, for a viewport size greater than `mobile` (up to 768px), these `<Level.Item>`s will be stacked.

<Playground>
  <Level breakpoint="fullhd">
    <Level.Item textAlign="centered">
      <div>
        <Heading>Tweets</Heading>
        <Title as="p">3,210</Title>
      </div>
    </Level.Item>
    <Level.Item textAlign="centered">
      <div>
        <Heading>Following</Heading>
        <Title as="p">210</Title>
      </div>
    </Level.Item>
    <Level.Item textAlign="centered">
      <div>
        <Heading>Followers</Heading>

      </div>
    </Level.Item>
    <Level.Item textAlign="centered">
      <div>
        <Heading>Likes</Heading>
        <Title as="p">321K</Title>
      </div>
    </Level.Item>
  </Level>
</Playground>
